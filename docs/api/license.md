# 授权特征码 API

## 接口列表

| 接口路径 | 请求方法 | 功能描述 |
|---------|----------|----------|
| /api/license | GET | 获取授权特征码 |

---

## GET /api/license

**功能描述**：获取服务器物理网卡的授权特征码列表。系统会自动过滤以下网卡：
- 本地回环网卡（loopback）
- Docker 网卡（名称包含 `docker`）
- 虚拟网卡（`veth`、`virbr`、`dummy`、`tun`、`tap` 等）
- 未启用或没有硬件地址的网卡
- 无有效 IP 地址的网卡

特征码算法：对每张有效物理网卡，取其 MAC 地址（大写十六进制，`-` 分隔），拼接盐值 `AAS` 后进行 SHA-256 哈希，取哈希结果的低 32 位有符号整数转换为十进制字符串，并加上前缀 `SZTY`。

**请求参数**：无

**响应格式**：JSON 数组

```json
[
  {
    "code": "SZTY1234567890",
    "interface": "eth0",
    "ip": "192.168.1.10"
  },
  {
    "code": "SZTY9876543210",
    "interface": "eth1",
    "ip": "10.0.0.5"
  }
]
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| code | string | 授权特征码，前缀为 `SZTY` |
| interface | string | 网卡名称 |
| ip | string | 网卡对应的 IP 地址（优先 IPv4） |

**错误响应**：

```json
{
  "error": "Error message"
}
```
